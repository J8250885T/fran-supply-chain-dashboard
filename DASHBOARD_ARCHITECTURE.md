## ARCHITECTURE OVERVIEW

The Fran dashboard is a **static web app backed by a JSON data file** — it is NOT a live API. It renders from `data.json`, which is generated by two parallel pipelines:

```
Pipeline A (Local Python server)
├── fran_ingest.py        → fran.db (SQLite events)
├── fran_market.py        → live market inputs (Brent, WTI, AUD/USD, BDI, fuel prices)
├── fran_db.py            → aggregates db events → data.json (subset)
└── server.py             → serves dashboard.html + data.json on :8080

Pipeline B (Cloudflare Pages / Netlify static deploy)
├── Dashboard/fran_push.sh  → pushes local data.json to GitHub
├── Dashboard/index.html    → redirects to fran-supply-chain.pages.dev
└── GitHub Pages/CDN        → serves dashboard globally
```

### Key Files

| File | Purpose |
|---|---|
| `Dashboard/dashboard.html` | The main UI — dark-themed, monospace, reads data.json via JS fetch |
| `Dashboard/data.json` | The single source of truth for the UI — all metrics, events, reserves, shipments, inflation model |
| `Dashboard/server.py` | Local Python HTTP server — runs on Mac Mini, serves dashboard at :8080 |
| `Dashboard/fran_push.sh` | Pushes data.json to GitHub → Cloudflare Pages auto-deploys |
| `fran_market.py` | Fetches live Brent/WTI/AUD-USD/fuel prices, writes to fran.db and data.json |
| `fran_ingest.py` | RSS feed scraper — pulls events from ~55 sources into fran.db |
| `fran_db.py` | SQLite query tool — context, last-run, check-url, add-event, update-state, log-report |
| `Memory/sources.json` | Registry of all RSS/API sources by tier and region |

## HOW DATA FLOWS

### 1. Market Data (live, on demand or cron)
```
fran_market.py (cron every 6h)
  → writes to fran.db (liveInputs table)
  → writes to data.json (liveInputs section)
```

### 2. Event Ingestion (batch, every 6h)
```
fran_ingest.py --tier 3 4 --method rss (cron every 6h)
  → deduplicates against fran.db
  → writes new events to fran.db (events table)
```

### 3. Data.json Assembly (on each dashboard page load)
```
server.py: do_GET(/data.json)
  → calls parse_memory_files()
  → reads Memory/*.md (fuel-markets, food-supply, shipping-routes, australia-fuel, daily-log)
  → reads fran.db (most recent events + current state)
  → updates data.json servoOutages, reserves, lastUpdated
  → returns data.json to dashboard
```

### 4. Public Dashboard (Cloudflare Pages)
```
fran_push.sh (cron after each update)
  → commits data.json to GitHub
  → Cloudflare Pages auto-builds → fran-supply-chain.pages.dev
```

## WHAT data.json CONTAINS

The file has 7 top-level sections:

| Section | Contents | Updated By |
|---|---|---|
| `lastUpdated` | Timestamp | server.py on every request |
| `warAlert` | Latest fuel/food/geo events, risk level, watch items | fran.db (events) |
| `servoOutages` | Total + by-state station outage count | fran_market.py / server.py |
| `shipments` | Named vessels, ETAs, routes, map routes | fran_market.py |
| `refineries` | South Korea / Singapore / Taiwan / Japan utilisation % | fran_market.py |
| `reserves` | Diesel / petrol / jet fuel days + government actions | fran_market.py |
| `shippingRoutes` | Hormuz / Bab el-Mandeb / Suez / Malacca status | fran_market.py |
| `bankEmpty` | Household tier model (6 tiers), government fiscal buffer | fran_market.py |
| `inflation` | CPI tiers, state breakdown, weekly items, liveInputs, modelInputs | fran_market.py |
| `dbLastUpdated` | When fran.db was last updated | fran_ingest.py |
| `dbEvents[]` | Last ~100 events from fran.db | fran_db.py |

## KNOWN ISSUES

### 1. Dashboard Data Is Stale
**Problem:** `data.json` is seeded from a static file dated 2026-03-31. Crons do write to it, but the Cloudflare Pages version may not reflect the latest data if `fran_push.sh` hasn't run recently.

**Fix:** Run `fran_push.sh` manually after each major update, or schedule it via cron.

### 2. server.py Race Condition
**Problem:** `parse_memory_files()` in `server.py` tries to read Memory/*.md AND update data.json on every `/data.json` request. This works locally but the Cloudflare Pages version reads from the git-committed `data.json` — not the live one being updated by server.py.

**Fix:** The public dashboard reads the git-committed version. You need to push after each server.py update cycle.

### 3. servoOutages Total vs State Mismatch
**Problem:** `servoOutages.total` is 274 (from fuel-markets.md parsing), but `servoOutages.byState` sums to 851. The regex extractor for `total` may be picking up an older figure. The dashboard uses `total` as the headline number.

**Fix:** Normalise the byState figures or write a proper servo tracker that writes structured data to memory rather than relying on regex extraction.

### 4. Cron Does NOT Update the Cloudflare Dashboard
**Problem:** The cron jobs (war-fuel-food, servo-tracker, asian-refinery, shipment-validator) write to fran.db and memory files. They do NOT run `fran_push.sh`. So the public dashboard lags until someone manually pushes.

**Fix:** Add `fran_push.sh` as a final step in the cron jobs, OR wire a GitHub Actions workflow that polls fran.db and rebuilds.

## QUICK REFERENCE COMMANDS

```bash
# Serve locally
python3 Dashboard/server.py

# Push to Cloudflare Pages
bash Dashboard/fran_push.sh

# Run ingestion (all RSS feeds)
python3 Memory/fran_ingest.py --tier 3 4 --method rss

# Check last cron run
python3 Memory/fran_db.py last-run --cron war-fuel-food

# Get recent events
python3 Memory/fran_db.py context --days 3

# Check a URL
python3 Memory/fran_db.py check-url --url "https://..."

# Force data.json rebuild from memory files
# (hit the refresh endpoint while server.py is running)
curl http://localhost:8080/refresh
```

## WHAT THE DASHBOARD SHOWS

The UI has these panels:
1. **War Alert feed** — scrolling list of recent fuel/food/geo events with source + confidence
2. **Servo Outage Map** — Australia states, outage count per state
3. **Shipment Tracker** — named vessels with ETA, route style (solid/dashed), origin→destination
4. **Chokepoint Status** — Hormuz, Bab el-Mandeb, Suez, Malacca with traffic status
5. **Asian Refinery Utilisation** — South Korea / Singapore / Taiwan / Japan gauges
6. **Australian Reserves** — Diesel / Petrol / Jet days with rationing red line
7. **Household Impact** — 6 household tiers with monthly deficit and runway
8. **Inflation Model** — national CPI tier, state breakdown, weekly items chart, live Brent/WTI/AUD
